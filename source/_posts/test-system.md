####重装系统后的hexo博客测试

重装系统后，本博文用于测试迁移后的博客系统能否正常工作，按照预想是可以正常工作的，而且本质上可以对git系统的使用和博客迁移做一个总结：

* git的默认分支才是真正的核心，clone、push、pull的操作对象都是默认分支，而不一定是master分支。
* hexo在部署时可以借由config文件指定[生成的静态网页文件]推送的git仓库和分支。正是由此才将hexo博客的原始文件和编译后的静态文件存储于同一仓库的不同分支。
* hexo在更换电脑或者重装系统或者在其他电脑部署时的操作如下：
    1. `git clone git@github.com:....`克隆目标仓库（因为博客源文件时默认分支，所以本质上是拷贝了源文件）
    2. 进入克隆下来的文件夹执行`npm install` 安装相应的依赖（因为克隆下来的文件夹里已经有hexo init命令操作历史，**所以这里不再执行**）
    3. 在克隆下来的文件夹里执行`npm install hexo-deployer-git --save`安装hexo的部署插件

    说明：此处默认系统已安装git、nodejs、hexo并完成与github相关的配置。
